# 腾讯云 IM 两日开发清单

## 一、腾讯云回调接入

- 新建独立 `imcallback` 模块，与原有业务代码隔离。
- 接入腾讯云新测试应用 `20044723`。
- 建立统一回调入口，完成 Token、AppID 和请求格式校验。
- 回调事件幂等落库，避免腾讯重复推送导致重复处理。
- 保存原始 JSON、事件类型、账号、群组、状态、TraceID、错误和耗时。
- 接入 Redis 队列、Worker 消费、失败重试和数据库补偿机制。
- 未实现但合法的事件记录为“已忽略”，避免腾讯持续重试。

### 已覆盖回调

- 消息：单聊/群聊发送前后、已读、撤回、发送异常。
- 群组：创建、入群、退群、解散、群满、群资料变更、成员资料变更、群主变更。
- 关系链：添加/删除好友、好友回应、加入/移除黑名单。
- 用户资料：头像、昵称等资料变更。
- 在线状态：登录、退出和状态变化。

### 数据投影

建立独立 IM 状态表，不写入旧业务表：

- 群组及群成员状态。
- 好友与黑名单关系。
- 用户资料。
- 在线状态。
- 腾讯账号与业务 UID 映射。

## 二、后台回调管理

后台菜单整合为：

```text
系统管理
└─ IM运维
   ├─ 运行概览
   ├─ 异常中心
   ├─ 回调日志
   └─ IM状态
```

### 运行概览

- 回调接收量、处理成功率、失败和死信数量。
- 重复推送、待处理积压和处理耗时。
- Worker、Redis 和队列健康状态。
- 回调趋势及事件类别分布。

### 回调日志

- 按时间、事件类型、状态、账号、群 ID 和 TraceID 筛选。
- 查看处理时间线、账号映射和错误信息。
- 受控查看腾讯原始 JSON。
- 区分正常、待处理、失败、死信和已忽略事件。

### 异常中心

- 展示待处理、超时、重试中和死信事件。
- 支持单条人工重新处理。
- 记录管理员操作审计。
- 防止旧事件覆盖较新的 IM 状态。

### IM 状态

- 查看群组、群成员、好友、黑名单、用户资料和在线状态。
- 支持业务 UID、IM UserID、群 ID 和状态查询。
- 可从状态记录追溯对应回调事件。
